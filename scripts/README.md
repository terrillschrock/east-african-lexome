# Scripts

This folder contains utility scripts for maintaining the EALex dataset.

The current workflow is **Model A**: the master spreadsheet (`African_Lexome_CLDF_Template.xlsx`) is the working source of truth, and the CLDF files in `cldf/` are exports generated from it. These scripts handle the export step.

---

## `regenerate_cldf.py`

Reads the master Excel spreadsheet and writes fresh CLDF-formatted files into the repository's `cldf/` folder. Run this whenever meaningful changes have been made to the spreadsheet that should be reflected in the public dataset.

### What it does

The script:

1. Opens the master spreadsheet
2. Reads each working sheet (LanguageTable, ParameterTable, FormTable, CognateTable, CognatesetTable, OrthoRegistry, SourceTable)
3. Writes each as a CLDF-compliant CSV file in `cldf/`:
   - `languages.csv`
   - `parameters.csv`
   - `forms.csv`
   - `cognates.csv`
   - `cognatesets.csv`
   - `orthographies.csv`
4. Converts the SourceTable into BibTeX format (`sources.bib`)
5. Cleans out placeholder/example rows from the FormTable

The schema file (`Wordlist-metadata.json`) is **not** regenerated by this script. If you need to modify the schema (e.g. adding a new column to a CLDF table), edit `cldf/Wordlist-metadata.json` directly.

### One-time setup

Before first use:

1. **Install Python 3** if not already installed (python.org)
2. **Install the openpyxl library**:
   ```
   py -m pip install openpyxl
   ```
   (Use `python3 -m pip install openpyxl` on Mac/Linux.)
3. **Edit the two paths at the top of `regenerate_cldf.py`** to match your machine:
   - `SPREADSHEET` — full path to your master Excel file
   - `REPO_CLDF_FOLDER` — full path to the `cldf/` folder of your local repo clone

### Running the script

From the command line, in this folder:

```
py regenerate_cldf.py        # Windows
python3 regenerate_cldf.py   # Mac/Linux
```

The script prints a row count for each generated file, so you can confirm the data flowed through correctly.

### After running

1. Open GitHub Desktop. You'll see modified files in `cldf/`.
2. Review the diffs to confirm the changes look right.
3. Commit with a descriptive message (e.g. "Update CLDF: integrate RVN concept contributions").
4. Push to GitHub.

### Troubleshooting

**"openpyxl is not installed"** — Run the pip install command in the setup section above.

**"Spreadsheet not found"** — Edit the `SPREADSHEET` path at the top of the script. Use double backslashes or forward slashes on Windows (`C:/Users/...` or `C:\\Users\\...`).

**"Repo cldf folder not found"** — Edit the `REPO_CLDF_FOLDER` path. This should point to the `cldf/` folder *inside* your local repo clone, not the repo root.

**Special characters look broken in the CSVs** — Ensure your spreadsheet is saved in a Unicode-aware format. The script writes UTF-8 with BOM-free encoding, which is the CLDF standard.
